# 知识库 22

## 标题
如何删除 Access 2010/2007 对不存在窗体的引用?

## 问题描述
针对提示窗体名称“xxx”拼写有误或引用了一个不存在的窗体。如果无效窗体名称是在宏中，在单击“确定”按钮之后，“操作失效”对话框将显示宏名称和宏参数。可打开“宏”窗口并输入正确的窗体名称。这种情况，本文将指导删除 Access 2010/2007 对不存在窗体的引用的方法。

## 分类
主类别: 预装软件
子类别: Office软件

## 关键词
Access 2010, Access 2007, 窗体, 引用

## 元信息
创建时间:2024-12-15|版本:1.0

## 解决方案
**故障现象：**

我用 Access 的“**资产.accdb**”数据库模板建立了一个数据库，并且将这个模板的大部分表格和窗体都删除或重新命名了。现在每当我打开这个数据库文件时，都得到如下提示：

窗体名称“xxx”拼写有误或引用了一个不存在的窗体。如果无效窗体名称是在宏中，在单击“确定”按钮之后，“操作失效”对话框将显示宏名称和宏参数。可打开“宏”窗口并输入正确的窗体名称。

我没有设置任何宏，也不清楚原来的“**资产.accdb**”数据库模板具体在什么地方引用了“xxx”窗体，怎么样才能彻底消除这个提示呢？

**解决方案：**

打开 Access，在“**文件**”菜单上单击“**选项**”。

![](https://webdoc.lenovo.com.cn/lenovowsi/20120604/1338798361484_478.png)

在弹出的“**Access 选项**”窗口左侧，点击“**当前数据库**”。

![](https://webdoc.lenovo.com.cn/lenovowsi/20120604/1338798361484_432.png)

从右边的窗格中找到“**显示窗体**”一栏，点击下拉菜单，选择一项确认存在的窗体。

![](https://webdoc.lenovo.com.cn/lenovowsi/20120604/1338798361484_376.png)

点击“确定”按钮，退出“**Access 选项**”窗口。接受弹出的提示，退出 Access。

![](https://webdoc.lenovo.com.cn/lenovowsi/20120604/1338798361484_313.png)

重新打开有问题的数据库文件，原来恼人的提示不见啦。
