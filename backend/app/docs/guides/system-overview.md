# ITS 系统业务介绍

## 系统是什么

**ITS（Intelligent Technical Service）** 是为联想构建的 AI 客服系统，核心解决两个问题：

> **"我的设备坏了，怎么修？"** 和 **"去哪里修？怎么去？"**

用户通过对话界面提问，系统自动识别意图、调度智能体、检索知识、返回答案，全程流式输出。

---

## 两大子系统

### 1. ITS 智能客服（多智能体中枢）

用户对话的入口，负责理解用户意图并协调各方资源给出答案。

**能回答的问题类型：**

| 场景 | 示例 | 处理方式 |
|------|------|---------|
| 技术故障诊断 | "蓝屏代码 0x000001 怎么解决" | 查私域知识库 + Web 搜索兜底 |
| 信息搜索 | "最新的联想驱动在哪下载" | 实时 Web 搜索 |
| 服务站查询 | "附近有没有联想维修点" | 查 MySQL 服务站数据库 |
| 导航引导 | "导航去最近的维修站" | 百度地图 MCP 路径规划 |
| 日常闲聊 | "你好" | 直接回复 |

### 2. ITS 知识库平台（RAG 系统）

私域知识的存储和检索引擎，是智能客服的"大脑记忆"。

**核心能力：**
- 上传联想内部维修文档 → 自动向量化入库
- 语义检索：用户问"蓝屏"能匹配到"系统崩溃"相关文档
- 从联想官方知识库爬取文章，持续扩充知识

---

## 架构哲学

系统采用**双引擎 + 职责分离**设计：

```
用户
 ↓
多智能体中枢（只做决策，不存知识）
 ├─ 调度智能体  → 识别意图，分发任务
 ├─ 技术顾问    → 调知识库/搜索，回答"怎么修"
 └─ 业务助手    → 查数据库/地图，回答"去哪修"
         ↓
知识库平台（只管问答，不碰业务）
```

**核心原则：** 中枢不存知识只做决策，知识平台不碰业务只管问答。每个智能体各司其职，边界清晰，互不越权。

---

## 面向用户

联想的 **C 端用户**（购买了联想产品的消费者），当设备出现问题时，通过这套系统获得：
1. 自助故障诊断和修复指南
2. 就近服务站查询和导航
